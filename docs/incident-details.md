# 🚨 Incident Details (Timeline-Based)

## 📌 Overview

This section describes the real-world incident in a structured timeline format.

---

## 🕒 Timeline of Events

### 🔹 Day 1 – Initial Infection

* A student downloaded a file from an unknown source
* The file appeared normal and harmless
* The file was extracted on an Android device
* The file contained a hidden Remote Access Trojan (RAT)

---

### 🔹 Day 1 – Malware Execution

* After extraction, the RAT silently executed in the background
* No visible signs of infection were noticed
* User assumed everything was normal

---

### 🔹 Day 1 – Suspicious Activity

* At night, abnormal device behavior was observed
* Sudden high data usage detected
* College Wi-Fi (5GB limit) was completely consumed

---

### 🔹 Day 2 – Investigation Attempt

* User approached a local cyber café
* The issue was identified as critical
* They refused to handle due to potential system risk

---

### 🔹 Day 2 – Escalation

* The case was brought for a secure solution
* Objective: Recover data safely without spreading infection

---

### 🔹 Day 2 – Resolution

* Live Linux environment used
* Data extracted securely
* No infection spread to host system

---

## 📌 Conclusion

This incident highlights how simple user actions (like extracting unknown files) can lead to serious cybersecurity threats.
